# L3_Clustering_MonteCarlo.ipynb 问题分析报告

## 问题 1：`gen_panel` 中 AR(1) 误差生成有重复/冲突代码（严重 Bug）

**位置**：第 100-104 行

```stata
* AR(1) 误差：eps_it = rho * eps_{i,t-1} + u_it
sort unit t
gen eps = rnormal(0, 1) if t == 1
replace eps = `rho' * L.eps + rnormal(0, 1) if t > 1 & !missing(L.eps)
xtset unit t
replace eps = `rho' * eps[_n-1] + rnormal(0, 1) if t > 1 & unit == unit[_n-1]
```

**问题**：
- 第 102 行用 `L.eps`（时间序列滞后算子）——但此时尚未 `xtset`，`L.eps` 会报错或产生不可预期的结果。
- 第 103 行 `xtset unit t` 放在第一次 `replace` 之后，顺序错误。
- 第 104 行又用 `eps[_n-1]` 做第二次 `replace`，覆盖了第 102 行的结果，但逻辑重复。
- 实际上第 102 行和第 104 行做了同一件事，第 102 行是多余的且可能报错。

**修复方案**：删除第 102 行，将 `xtset` 移到第 101 行之前，只保留第 104 行的正确写法：

```stata
sort unit t
xtset unit t
gen eps = rnormal(0, 1) if t == 1
replace eps = `rho' * eps[_n-1] + rnormal(0, 1) if t > 1 & unit == unit[_n-1]
```

## 问题 2：`gen_panel` 中时间效应生成逻辑有误

**位置**：第 93-97 行

```stata
* 时间效应
gen time_fe = rnormal(0, 0.5) if t == 1
bysort t: egen tfe = mean(time_fe)
drop time_fe
rename tfe time_fe
```

**问题**：
- 首先生成 `time_fe` 但只在 `t == 1` 时有值（其他为缺失）。
- 然后 `bysort t: egen tfe = mean(time_fe)` 对每个 t 取均值——但只有 t=1 时有非缺失值，所以 t=2..10 的 `tfe` 全是缺失值！
- 这意味着时间效应实际上**只有第一期有效**，后面 9 期全是缺失值，导致 `Y` 生成时 `time_fe` 对 t>=2 的观测贡献为缺失。

**修复方案**：应该为每个时间期独立生成时间效应：

```stata
* 时间效应：每个时期独立生成
bysort t: gen time_fe = rnormal(0, 0.5) if _n == 1
bysort t: egen tfe = mean(time_fe)
drop time_fe
rename tfe time_fe
```

或者更简洁：

```stata
* 时间效应
gen time_fe = .
forvalues tt = 1/`t' {
    replace time_fe = rnormal(0, 0.5) if t == `tt'
}
```

## 问题 3：MC 循环中 p 值计算使用了错误的自由度

**位置**：第 174、177、180 行（以及第 235、237 行）

```stata
local p_iid = 2 * ttail(e(df_r), abs(_b[D] / _se[D]))
```

**问题**：
- `reghdfe` 在吸收(absorb)固定效应后，`e(df_r)` 返回的是残差自由度，已经扣除了吸收掉的固定效应。
- 但对于聚类标准误，t 检验应该使用 **G - 1**（聚类数减 1）作为自由度，而不是 `e(df_r)`。
- 对于 HC1 稳健标准误，也应该使用更保守的小样本校正。
- 这会导致 p 值略微偏小（拒绝率略微偏高），虽然对定性结论影响不大，但不够精确。

**修复方案**（针对聚类标准误）：

```stata
reghdfe Y D, absorb(unit t) vce(cluster unit)
local G = e(N_clust)   // 聚类数
local p_cl = 2 * ttail(`G' - 1, abs(_b[D] / _se[D]))
```

## 问题 4：第 4 节 MC 中 `quietly` 块内的 `display` 不会输出

**位置**：第 183 行

```stata
if mod(`r', 50) == 0 display "完成 `r' / `nreps'"
```

**问题**：
- 这行在 `quietly { }` 块内部，所以 `display` 被静默了，用户看不到进度。
- 实际上用户运行时会感觉卡住了，不知道跑了多少。

**修复方案**：将进度显示移到 `quietly` 块外面：

```stata
quietly {
    gen_panel, n(500) t(10) rho(0.8) te(0.0) seed(`r')
    ...
}
post mcresults (`r') (`p_iid') (`p_hc') (`p_cl')
if mod(`r', 50) == 0 display "完成 `r' / `nreps'"
```

## 问题 7：第 7 节两维聚类中 `vce(cluster unit t)` 语法可能不兼容

**位置**：第 302 行

```stata
reghdfe Y D, absorb(unit t) vce(cluster unit t)
```

**问题**：
- `reghdfe` 的两维聚类语法是 `vce(cluster id1 id2)`，但这里 `t` 同时也是被吸收的固定效应变量。
- 当 `t` 既在 `absorb()` 中又在 `vce(cluster)` 中时，`reghdfe` 可能会报错或行为异常。
- 更安全的做法是创建一个时间聚类变量的副本。

**修复方案**：

```stata
gen time_cluster = t
reghdfe Y D, absorb(unit t) vce(cluster unit time_cluster)
```

## 问题 8：第 6 节 Wild Bootstrap 中 `boottest` 的 `cluster(unit)` 与 `reghdfe` 的聚类冲突

**位置**：第 275 行

```stata
boottest D, boottype(wild-t) reps(999) cluster(unit)
```

**问题**：
- `boottest` 的 `cluster(unit)` 选项指定 bootstrap 重抽样在 `unit` 层面进行。
- 但前面的 `reghdfe` 已经用了 `vce(cluster unit)`，`boottest` 需要确保它使用的是正确的基础回归结果。
- 如果 `reghdfe` 的估计结果没有被正确传递，`boottest` 可能使用错误的方差矩阵。
- 建议先用 `estimates store` 保存，再显式调用。

## 问题 9：`gen_panel` 中 `seed` 参数默认值为 0，但 `set seed 0` 在 Stata 中无效

**位置**：第 77、79 行

```stata
syntax, [n(integer 500) t(integer 10) rho(real 0.8) te(real 0.0) seed(integer 0)]
...
set seed `seed'
```

**问题**：
- Stata 中 `set seed 0` 不会报错，但种子为 0 时随机数生成器的行为可能在不同 Stata 版本间不一致。
- 更安全的做法是使用非零默认种子，或者要求用户显式指定。

## 问题 10：`gen_panel` 中 `treated` 的生成方式导致处理组比例不稳定

**位置**：第 84-87 行

```stata
gen u = runiform()
sort u
gen treated = (_n <= `n' / 5)
```

**问题**：
- 当 `n` 不能被 5 整除时（如 `n=30`），`n/5 = 6`，没问题。
- 但当 `n` 较小时，处理组和控制组的比例可能不够均衡，影响统计推断。
- 更稳健的方式是用 `_n <= ceil(`n'/5)` 或直接使用 `runiform() < 0.2`。
